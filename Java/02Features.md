# 1. Simple

Java is considered a simple programming language because it removes many complex features found in languages like C and C++.

### Why is it Simple?

* Easy to learn and understand.
* Uses simple syntax similar to C/C++.
* No pointers for direct memory access.
* Automatic memory management through Garbage Collection.
* Rich Standard Library.

### Example

```java
int age = 20;
System.out.println(age);
```

---

# 2. Object-Oriented

Java is based on the concept of **Objects** and **Classes**.

Everything in Java is designed around Object-Oriented Programming (OOP), which helps in writing modular, reusable, and maintainable code.

### Main OOP Principles

* Encapsulation
* Inheritance
* Polymorphism
* Abstraction

### Example

```java
class Student {

    String name = "Amay";

    void display() {
        System.out.println(name);
    }

}
```

# 3. Platform Independent

Java follows the principle:

> **Write Once, Run Anywhere (WORA)**

A Java program is compiled into **bytecode**, which can run on any operating system that has a JVM.

### Process

```text
Source Code
    ↓
Bytecode (.class)
    ↓
JVM
    ↓
Machine Code
```

### Benefit

The same Java program can run on:

* Windows
* Linux
* macOS

without changing the source code.

---

# 4. Architecture Neutral

Java programs are not dependent on the processor architecture.

The generated bytecode can run on any CPU architecture as long as a compatible JVM is available.

Examples of processor architectures:

* x86
* x64
* ARM

---

# 5. Portable

Java applications can be easily moved from one system to another.

Reasons:

* Fixed primitive data type sizes.
* Platform-independent bytecode.
* Standard Java libraries.

---

# 6. Secure

Java provides several security features.

### Security Features

* No pointers.
* Bytecode verification.
* Class Loader.
* Security Manager (legacy feature).
* Exception Handling.
* Automatic memory management.

These features reduce security risks such as unauthorized memory access.

---

# 7. Robust

A robust program continues to work correctly even when unexpected situations occur.

Java is robust because it provides:

* Exception Handling
* Strong Type Checking
* Garbage Collection
* Automatic Memory Management

### Example

```java
try {

    int result = 10 / 0;

} catch (ArithmeticException e) {

    System.out.println("Cannot divide by zero.");

}
```

---

# 8. Multithreaded

Java supports multiple threads running simultaneously.

A thread is a lightweight process.

Multithreading improves:

* Performance
* CPU Utilization
* Responsiveness

### Example

```java
Thread t = new Thread();
t.start();
```

---

# 9. Distributed

Java can be used to develop distributed applications where different computers communicate over a network.

Examples:

* Client-Server Applications
* Web Services
* Distributed Systems

---

# 10. Dynamic

Java can load classes dynamically during runtime.

This allows applications to become more flexible and extensible.

Example:

* JDBC Drivers
* Reflection API

---

# 11. High Performance

Java is faster than traditional interpreted languages because of the **Just-In-Time (JIT) Compiler**.

The JIT compiler converts bytecode into machine code while the program is running.

This improves execution speed.

---

# 12. Automatic Garbage Collection

Java automatically removes objects that are no longer being used.

Benefits:

* Prevents memory leaks.
* Reduces programmer effort.
* Improves memory management.

Example:

```java
Student s = new Student();

s = null;
```

The unused object becomes eligible for Garbage Collection.

# Interview Notes

### Why is Java Platform Independent?

Because Java programs are compiled into **bytecode**, which is executed by the JVM instead of directly by the operating system.

---

### Difference Between Platform Independent and Portable

| Platform Independent                                   | Portable                                                                                     |
| ------------------------------------------------------ | -------------------------------------------------------------------------------------------- |
| Same bytecode runs on any operating system with a JVM. | Java programs can be easily transferred between different environments without modification. |

---

### What Makes Java Secure?

* Bytecode Verification
* Class Loader
* No Pointers
* Exception Handling
* Automatic Memory Management
